RELD: A Knowledge Graph of Relation Extraction Datasets

Introduction

In recent years, various supervised, semi-supervised, and unsupervised models have been proposed to extract relations from natural language texts. Consequently, various relation extraction datasets have been proposed to benchmark existing RE tools. These datasets are in different formats, e.g., NYT-FB, Wikipedia-Wikidata are in JSON, WEBNLG is in XML, SemEval 2010 Task 8 is in text form. Apart from this, these datasets have different style of annotation. The source of generation of these datasets also lack variance, which makes it hard for evaluations of RE systems. Keeping in view the aforementioned shortcomings, we present RELD, a single unified RDF representation of six relation extraction and classification datasets NYT-FB, Wikipedia-Wikidata, WEBNLG, SemEval 2010 Task 8, Google-RE and FewRel well-known public and freely accessible relation extraction datasets. In RELD, each relation is modelled as a unique RDF resource and attached various statistics/annotations (e.g, corresponding entities, sentences, or a number of tokens in the sentences etc.) as properties. The resulting RELD RDF knowledge graph consists of 55.54 million triples, 824 relations from 2 million sentences, collected from six datasets of different domains. To the best of our knowledge, RELD is the largest RDF dataset containing relations and entities annotations along with different information from diverse domains.

Some usful queires

Following are a few examples SPARQL queries on RELD endpoint.

Get all Relations from Google RE dataset.

SELECT DISTINCT ?relation
FROM <https://reld.dice-research.org/Google_RE>
WHERE {
?relation <https://reld.dice-research.org/schema/hasSentence> ?sent . }

Get all triples from Wikipedia-Wikidata

SELECT DISTINCT count(*)
FROM <http://reld.dice-research.org/Wiki-RE>
WHERE {
?s ?p ?o . }

Select all relations that have sentences which have number of token > 150

prefix reldv: <https://reld.dice-research.org/schema/>
SELECT DISTINCT ?r
where {
?r reldv:hasSentence ?o.
?o reldv:numOfTokens ?t.
Filter(?t > 150)
}

Select all sentences containing more than 2 named entities

Prefix reldv: <https://reld.dice-research.org/schema/>
SELECT DISTINCT ?sentence
Where{
?rId reldv:hasSentence ?sentence. ?sentence reldv:hasNamedEntity ?ne.
}
Group by ?sentence having (count(?ne) > 2)

Select relations with average number of subject position, object position

Prefix reld: <https://reld.dice-research.org/schema/>
SELECT DISTINCT ?rId (AVG(?subPos) as ?avgSubjPosition) (AVG(?objPos) as ?avgObjPosition)
where{
?rId reld:hasSentence ?sentence.
?sentence reld:subPos ?subPos.
?sentence reld:objPos ?objPos.
}

Select sentences with number of between tokens is higher than 10


Prefix reld: <https://reld.dice-research.org/schema/> SELECT DISTINCT ?sentence
where{
?rId reld:hasSentence ?sentence.
?sentence reld:reld:numBetToken ?bt.

} Group by ?sentence having(?nToken > 10)

Selecting 40 relations based on some filter criteria


Prefix reld: <https://reld.dice-research.org/schema/>
SELECT ?rId
WHERE {
?rId reldv:hasSentence ?sent .
?sent reldv:numOfTokens ?nT .
?sent reldv:numBetToken ?tB .
FILTER (?nT < 25 && ?tB > 5 )
} LIMIT 40

Selecting distinct relations based on number of sentences grouping


Prefix reld: <https://reld.dice-research.org/schema/>
SELECT DISTINCT ?rId ( AVG(?nToken) as ?avgToken ) (count(?ne) as ?avgNE)
WHERE {
?rId reldv:hasSentence ?sentence .
?sentence reldv:numOfTokens ?nToken .
?sentence reldv:hasNamedEntity ?ne .
} Group by ?rId having(count (?sentence ) = 700)

Usage and Technical details

Named Graphs

To use the data of a single relation extraction dataset use a named graph. Named graphs in RELD are kept using the https://reld.dice-research.org/$dataset_name$. $dataset_name$ in URI represents the name of each dataset. For simplicity, we use the names of the datasets as shown in Table.

Relations

Each relation in the parent dataset is considered individually in a URI structured as https://reld.dice-research.org/resource$R_ID$. $R_ID$ is a unique id for each relation. RDFS labels contain the labels information of the relation. In the latest version of RELD there is a total of 824 relations. If a relation in two datasets is the same, they are connected by owl:sameAs property. Also owl: equivalent property is used to connect those relations which have the same meaning but different representations. All the details of the relations are available online here

Sentences

Sentences are connected to respective labeled relations in the URI format as https://reld.dice-research.org/resource$S_DatasetName_ID$. Dataset name in $S_DatasetName_ID$ is the name present in Table. All the properties of the sentences are attached to the sentences. The detail of each property is available online in our schema.

Subject and Object

The subject represents the first entity in a sentence. In most cases, it is a named entity (PERSON, PLACE, DATE, etc,.) or nominal value. URIs used for subject and objects are https://reld.dice-research.org/resource$sub_DatasetName_ID$ and https://reld.dice-research.org/resource$obj_DatasetName_ID$ respectively.

Named Entity

Other named entities in a sentence are sometimes very useful especially training an ML algorithm. In RELD named entities are represented as https://reld.dice-research.org/resource$ne_FirstLetterOfDataset_ID$. In $ne_FirstLetterOfDataset_ID$ FirstLetterOfDataset comes from the dataset name, For example, for SemEval it is 's'.

Live SPARQL Endpoint

The live endpoint of RELD is available for querying anywhere on the internet. We have made it available on the Virtuoso open-source server. The endpoint is currently under maintenance it might result in slow query execution. In the meanwhile, you can also run a local instance of the endpoint. The details of running the local instance are available on our Github README file . Running two simple commands, and your local endpoint in Virtuoso is running. The prepared endpoint is available here in zip format.

Source Code

Source code for converting RE datasets to RDF is available on our Github repository . The requirments for running the code, and the basic commond to rund the code are given on the README file.

Static Dumps

The first version of the dumps is available for downloads. The dumps are in .ttl format. The ttl files names are self-descriptive. Each file represents an individual dataset converted to RDF. Same file names can be used during querying on our live endpoint.

Dumps in JSON-LD

We have also provide dumps in JSON-LD format for non semantic web users. Other communities in NLP, less aware of RDF and Semantic web technologies can download RELD dumps in well known JSON format.

Sustainability

We have a clear plan for future maintenance of the RELD Knowledge graph. In the next step, we are incorporating document-based relation extraction into the RELD schema. It will add more classes and relations to the original schema. The document-based relation extraction dataset is available in the form of DocRED. We have access to the dataset, but we are waiting for the final consent of the author. In the second step, we will target the licensed/paid datasets such as TACRED. This step will add almost all types of existing RE datasets to the RELD knowledge graph.
